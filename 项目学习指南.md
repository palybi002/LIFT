# LIFT 项目学习指南

## 📚 项目概述

**LIFT (Learning from Leading Indicators)** 是一个用于多变量时间序列预测的深度学习框架，发表于 ICLR 2024。该项目从**领先-滞后关系**（Lead-Lag Relationships）的角度重新思考了多变量时间序列中的通道依赖性问题。

### 核心创新点

1. **领先-滞后关系建模**
   - 传统模型在同一时间步混合多变量信息，会引入来自滞后变量的过时信息
   - LIFT 动态识别领先指标（Leading Indicators）和领先步数（Leading Steps）
   - 通过移动对齐来缓解变量间的不对齐问题

2. **动态选择机制**
   - 快速评估所有变量对之间的关系
   - 动态选择最相关的领先指标子集
   - 处理通道依赖性中的分布偏移

3. **参数高效的基线模型 LightMTS**
   - 参数量接近 DLinear
   - 性能大幅优于 DLinear
   - 在多个基准数据集上达到 SOTA

## 🔑 核心文件详解

### 1. `run_longExp.py` - 主执行脚本

**作用**：项目的主入口，负责解析命令行参数、初始化模型、加载数据、运行训练和测试。

**关键参数**：
- `--model`: 模型名称（DLinear, Autoformer, PatchTST 等）
- `--dataset`: 数据集名称（Weather, ETTh1, AirQuality 等）
- `--lift`: 是否使用 LIFT 增强
- `--seq_len`: 输入序列长度
- `--pred_len`: 预测序列长度
- `--leader_num`: 领先指标数量
- `--state_num`: 变量状态数量
- `--pretrain --freeze`: 使用预训练冻结的骨干网络

**示例**：
```bash
python run_longExp.py --dataset Weather --model DLinear --lift \
  --seq_len 336 --pred_len 96 --leader_num 4 --state_num 8 \
  --learning_rate 0.0005
```

### 2. `models/LIFT.py` - LIFT 核心模块

**关键类**：

#### `Model` 类
- LIFT 的包装器，接收一个骨干网络（backbone）
- 将骨干网络的预测结果传递给 LeadRefiner 进行细化

#### `LeadRefiner` 类
- **核心功能**：使用领先指标细化预测
- **输入**：
  - `x`: 历史序列 [B, C, L]
  - `y_hat`: 骨干网络预测 [B, C, H]
  - `leader_ids`: 领先指标 ID
  - `shift`: 领先步数
  - `r`: 相关性分数
- **处理流程**：
  1. 实例归一化（Instance Normalization）
  2. 获取移动后的领先指标序列
  3. 使用 FilterFactory 生成自适应滤波器
  4. 在频域中混合领先指标信息
  5. 返回细化后的预测

#### `FilterFactory` 类
- 生成基于状态的自适应滤波器
- 根据输入序列和相关性分数动态生成滤波器参数

**技术亮点**：
- 使用频域处理（FFT/IFFT）提高效率
- 动态滤波器生成，适应时变的领先关系
- 实例归一化缓解分布偏移

### 3. `models/DLinear.py` - DLinear 基线模型

**核心思想**：
- 将时间序列分解为趋势（Trend）和季节性（Seasonal）成分
- 对两个成分分别应用线性层进行预测
- 最后合并两个预测结果

**关键组件**：
- `series_decomp`: 使用移动平均分解时间序列
- `Linear_Seasonal`: 季节性成分的线性预测层
- `Linear_Trend`: 趋势成分的线性预测层

### 4. `exp/exp_main.py` - 实验主逻辑

**关键方法**：

- `_build_model()`: 构建模型
  - 动态导入模型类
  - 应用归一化（如果需要）
  - 加载检查点（如果有）
  - 包装 LIFT 框架
  - 处理分布式训练（DDP）

- `train()`: 训练循环
  - 早停机制（Early Stopping）
  - 学习率调度（Learning Rate Scheduling）
  - 混合精度训练（AMP）
  - 验证损失监控

- `vali()`: 验证函数
  - 在验证集上评估模型
  - 计算平均损失

- `test()`: 测试函数
  - 在测试集上评估模型
  - 计算 MAE、MSE、RMSE、MAPE、MSPE 等指标
  - 保存预测结果

### 5. `data_provider/data_loader.py` - 数据加载器

**支持的数据集**：
- ETT 系列（ETTh1, ETTh2, ETTm1, ETTm2）
- Weather
- ECL (Electricity)
- Traffic
- AirQuality
- 等等

**关键类**：
- `Dataset_ETT_hour`: ETT 数据集
- `Dataset_Custom`: 自定义 CSV 数据集
- 每个数据集类提供：
  - 数据加载和预处理
  - 标准化（StandardScaler）
  - 时间特征编码
  - 训练/验证/测试集划分

### 6. `util/lead_estimate.py` - 领先指标估计

**核心功能**：
- `estimate_indicator()`: 估计领先指标
  - 计算所有变量对之间的相关性
  - 识别最相关的 K 个领先指标
  - 估计最佳领先步数

- `shifted_leader_seq()`: 生成移动后的领先序列
  - 根据估计的领先步数移动领先指标
  - 对齐领先指标与目标变量

- `accurate_indicator()`: 精确计算相关性
  - 使用滑动窗口计算动态相关性

### 7. `settings.py` - 配置文件

**数据集配置**（`data_settings`）：
- 数据文件名
- 目标列
- 通道数量
- 预取批次大小

**模型超参数**（`hyperparams`）：
- 不同模型在不同数据集上的最佳超参数
- 学习率、批次大小、dropout 等

**辅助变量**：
- `need_x_y_mark`: 需要时间标记的模型列表
- `need_x_mark`: 需要输入时间标记的模型列表

## 🔬 算法原理深入

### LIFT 工作原理

1. **预处理阶段**（`run_prefetch.py`）：
   - 对整个数据集，计算所有时间步的领先-滞后关系
   - 保存领先指标 ID、领先步数、相关性分数
   - 保存到 `prefetch/` 目录以避免重复计算

2. **训练阶段**：
   ```
   输入序列 x [B, L, C]
       ↓
   骨干网络预测 y_hat [B, H, C]
       ↓
   加载预计算的领先指标信息
       ↓
   LeadRefiner:
       - 实例归一化
       - 移动领先指标序列
       - FilterFactory 生成自适应滤波器
       - 频域混合（FFT → 滤波 → IFFT）
       ↓
   细化后的预测 [B, H, C]
   ```

3. **关键技术**：
   - **动态选择**：每个变量选择 K 个最相关的领先指标
   - **动态移动**：根据领先步数对齐领先指标
   - **频域处理**：使用 FFT/IFFT 高效处理长序列
   - **实例归一化**：缓解非平稳性和分布偏移

### 相关性计算

在 `lead_estimate.py` 中：

```python
# 计算变量 i 和 j 之间的相关性
# shift: 领先步数（正数表示 j 领先于 i）
correlation = compute_correlation(x[:, :, i], x[:, :, j], shift)
```

### 滤波器生成

在 `FilterFactory` 中：
- 输入：历史序列特征、相关性分数
- 输出：频域滤波器
- 机制：
  1. 状态分类（将变量分类到不同状态）
  2. 每个状态生成一组滤波器参数
  3. 根据相关性分数加权混合滤波器

## 🔧 关键配置说明

### 通用参数
- `--seq_len`: 输入序列长度（默认 336）
- `--label_len`: 标签序列长度（默认 48，用于 Transformer 系列）
- `--pred_len`: 预测长度（常用：24, 96, 192, 336, 720）
- `--features`: 预测类型
  - `M`: 多变量预测多变量
  - `S`: 单变量预测单变量
  - `MS`: 多变量预测单变量

### LIFT 特定参数
- `--lift`: 启用 LIFT 增强
- `--leader_num`: 领先指标数量（建议 4-8）
- `--state_num`: 状态数量（建议 4-8）
- `--temperature`: Softmax 温度（默认 1.0）
- `--pretrain`: 使用预训练骨干网络
- `--freeze`: 冻结骨干网络参数
- `--pin_gpu`: 将预计算数据固定在 GPU（加速但需要更多显存）

### 训练参数
- `--train_epochs`: 训练轮数（默认 10）
- `--batch_size`: 批次大小（默认 32）
- `--learning_rate`: 学习率（建议 0.0001-0.001）
- `--patience`: 早停耐心值（默认 3）
- `--lradj`: 学习率调度策略（type1, type2, type3, TST, constant 等）
- `--use_amp`: 使用混合精度训练

## 💡 使用建议

### 1. 快速开始
如果你刚开始使用这个项目：
1. 先训练一个简单的 DLinear 骨干网络
2. 然后添加 `--lift` 参数体验 LIFT 增强

### 2. 超参数调优
- `leader_num`: 对于少变量数据集（<20 变量）使用 4，多变量使用 8-16
- `state_num`: 通常设置为 4-8，更大的值可能导致过拟合
- `learning_rate`: LIFT 通常需要比骨干网络更大的学习率（约 5-10 倍）

### 3. 计算资源优化
- 使用 `--prefetch` 预计算可以大幅加速训练
- 使用 `--pin_gpu True` 可以加速，但需要足够的 GPU 显存
- 使用 `--pretrain --freeze` 可以减少训练时间

### 4. 调试建议
- 查看 `logs/` 目录的日志文件
- 使用 `--only_test` 只进行测试，不训练
- 使用较小的 `--train_epochs` 快速验证配置

## 🐛 常见问题

### Q1: CUDA out of memory
**解决**：
- 减小 `--batch_size`
- 设置 `--pin_gpu False`
- 减小 `--leader_num`

### Q2: 预计算文件未找到
**解决**：
- 先运行 `run_prefetch.py` 生成预计算文件
- 或者不使用 `--lift` 参数

### Q3: 模型性能不如预期
**检查**：
- 是否使用了正确的超参数（参考 `settings.py`）
- 是否完成了预训练
- 数据集是否正确加载和标准化
